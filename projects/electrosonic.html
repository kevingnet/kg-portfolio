Easy schedule is designed to use TCP/IP as the communications link. TCP/IP is a set of protocols that allow two computers (or other devices) to pass data between them. It was made popular by the explosive growth of the Internet for which the protocol was originally designed. The best way to understand the concepts surrounding TCP/IP is to think about the problem that it solves using an example. A typical network will have many different computers each running many different programs. Imagine you are sitting at a computer and wish to “talk” to a different computer. You need two pieces of information, the computer you wish to talk to and the program running on that computer. As an example from the Internet, you may wish to talk to Electrosonic’s main computer and retrieve a web page from the application that handles web page requests. In the language of TCP/IP the following notation is used… The computer is referred to using an IP address. The application is referred to using a PORT number. The computer that initiates the connection (Your Computer) is called a client. The computer that is connected to (Electrosonic’s server) is called a server. Every computer on a network has a unique IP Address and every application running on the same computer has a unique port number ( ranging from 1 to 65535 ). At the lowest level an IP address consists of 4 numbers between 1 and 255, written as in the following example 205.2.12.44. IP addresses can also be referenced in a more human readable form such as electrosonic.com. mnemonic names such as these are translated into their numeric equivalent by your computer. Easy scheduler acts as a client for as many servers as are appropriate for the installation. Using TCP/IP as the main communication method introduces a slight problem. How do we control RS232 based equipment such as projectors and source machines. Easy Schedule solves this problem using a very clever piece of equipment called a Port Server II. This box contains between 16 and 48 RS232 serial ports and a single TCP/IP Ethernet connection. By using this single ethernet connection Easy Schedule can talk to all 48 serial ports as if they were local to the computer. TCP/IP, and networks in general, provides huge benefits for distributed show control systems such as Easy Schedule. Locality is effectively removed as an issue, and since networks are inherently scalable the number of devices controlled is almost limitless. Below is an explanation of components and concepts used: 1.1 Product You can think of it as a network packet. SiteLinx uses products to share information in the network. A SiteLinx product is ‘wrapped’ in a new layer on top of TCP/IP protocol, which we have named ES4000 protocol. Applications wishing to use the services of SiteLinx must comply with the ES4000 protocol. A product has a type, source, destination and usually some data. A product can contain a message, a file or binary data. 1.2 Producer A producer is a node (computer or application) that is able to send a product to SiteLinx. 	int iNumberOfCues=GetSize();

	int iFirstCue = iCueStart;
	if( iFirstCue > iNumberOfCues ) iFirstCue = iNumberOfCues;

	int iLastCue = iFirstCue + iCueCount;
	if( iCueCount == -1 ) iLastCue = iNumberOfCues;
	if( iLastCue > iNumberOfCues ) iLastCue = iNumberOfCues;

	bool bChange;
	for( int i=iFirstCue; i<iLastCue; i++ )
	{
		CEvent* pE = GetEvent(i);
		bChange = false;
		if( pE ) {
			switch( iCueTimeTypes )
			{
			case TIME_TYPE_AT:
				if( pE->IsAtTime() )
					bChange = true;
				break;
			case TIME_TYPE_WAIT:
				if( pE->IsWaitTime() )
					bChange = true;
				break;
			case TIME_TYPE_HALT:
				if( pE->IsHaltTime() )
					bChange = true;
				break;
			case TIME_TYPE_HALT_UNTIL:
				if( pE->IsHaltUntilTime() )
					bChange = true;
				break;
			default:
				bChange = true;
				break;
			}
			if( bChange )
			{
				if( bAdd )
					pE->GetTimeRef() += time;
				else
					pE->GetTimeRef() -= time;
				if( pE->GetTime() < 0 )
					pE->SetTime(0);
				if( pE->GetTime() > 86399 )
					pE->SetTime(86399);
			}
		}
	}
}

//  StartComm()
//  Starts communications over the comm port.
//
//  StartComm makes sure there isn't communication in progress already,
//  the hNewCommFile is valid, and all the threads can be created.  It
//  also configures the hNewCommFile for the appropriate COMM settings.
//
//  If StartComm fails for any reason, it's up to the calling application
//  to close the Comm file handle.
//
//
BOOL CCommunicate::StartComm( HANDLE hNewCom )
{
	// Is this a valid comm handle?
	if (GetFileType( hNewCom ) != FILE_TYPE_CHAR) {
		TRACE("File handle is not a comm handle.\n");
		return FALSE;
	}

	// Are we already doing comm?
	if (m_ProcessControl.hCommFile != NULL) {
		TRACE("Already have a comm file open\n");
		return FALSE;
	}
	m_ProcessControl.hCommFile = hNewCom;

	// Setting and querying the comm port configurations.

	// Configure the comm settings.
	COMMTIMEOUTS commtimeouts;
	DCB dcb;
	COMMPROP commprop;
	DWORD fdwEvtMask;

	// These are here just so you can set a breakpoint
	// and see what the comm settings are.  Most Comm settings
	// are already set through TAPI.
	GetCommState(m_ProcessControl.hCommFile, &dcb);
	GetCommProperties(m_ProcessControl.hCommFile, &commprop);
	GetCommMask(m_ProcessControl.hCommFile, &fdwEvtMask);
	GetCommTimeouts(m_ProcessControl.hCommFile, &commtimeouts);

	// The CommTimeout numbers will very likely change if you are
	// coding to meet some kind of specification where
	// you need to reply within a certain amount of time after
	// recieving the last byte.  However,  If 1/4th of a second
	// goes by between recieving two characters, its a good 
	// indication that the transmitting end has finished, even
	// assuming a 1200 baud modem.
	commtimeouts.ReadIntervalTimeout         = 2;
	commtimeouts.ReadTotalTimeoutMultiplier  = 0;
	commtimeouts.ReadTotalTimeoutConstant    = 0;
	commtimeouts.WriteTotalTimeoutMultiplier = 0;
	commtimeouts.WriteTotalTimeoutConstant   = 0;

			szTemp = szTemp.Mid( 1, szTemp.Find( "\"", 1 ) - 4 ) + "mdb";

		WIN32_FIND_DATA fd;
		HANDLE hFile = FindFirstFile( szTemp, &fd );
		if ( hFile == INVALID_HANDLE_VALUE )

			FindClose( hFile );

		}
		FindClose( hFile );

		//create db object
		if ( m_pdb == NULL )
		{
			m_pdb = new CDaoDatabase;
		}
		//open database
		m_pdb->Open( (LPCTSTR)szTemp );
		return true;
	}    
	catch( CDaoException* e )    
	{
        TRACE( e->m_pErrorInfo->m_strDescription );
		e->Delete();
