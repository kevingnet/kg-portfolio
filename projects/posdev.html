Positive Developments

Network and Programming Support · Oct 1999 – Jan 2000 · Anaheim, CA

Positive Developments — network support and warehouse-industry programming for PalmOS and handheld devices.

  • SendGTL: Win32 MFC utility that loads firmware and data files to warehouse handhelds over serial — parameterized via SendGTL.txt with manufacturer/model lists, comm settings, and custom bitmap dialogs
  • Allstate Floral (PalmOS): order and item scanning on Palm/Symbol hardware — barcode scan screens, PDI file import, order index navigation (source tree: Allstate Floral V2)
  • Help desk and phone support for in-house staff and warehouse clients

Work Performed

Programmed warehouse applications for PalmOS and other handheld devices; maintained and extended the Allstate Floral order-entry scanner (v2.07 bug fixes and UI adjustments).

Built SendGTL — a configurable loader that reads pipe-delimited parameter rows, validates file paths, and drives serial transfer dialogs with registry persistence (Positive Developments registry key).

Provided network support and help desk coverage for employees and warehouse clients by phone.

TEXT SendGTL.txt SendGTL.txt 
//////////////////////////////////////////////////////////////////////////////////////////
//	This file is used by SendGTL.exe. SendGTL.exe is a utility that	gathers		//
//	parameters from this file and/or the user, and runs a program from the 		//
//	parameter list. Read the instructions below for usage:				//
//	Enter parameter lists to be given as options to the user on one line each,	//
//	Use the | symbol as a separator. that is the pipe symbol,			//
//	on the PC key above the back slash. Do not use the pipe 			//
//	symbol for any other purpose as it is the field separator.			//
//	The parameters format is as follows:						//
//											//	
//////////////////////////////////////////////////////////////////////////////////////////

[POSITION] [PARAMETER NAME]	[FORMAT]				[REQUIRED]

1)	Manufacturer		Text					Yes
2)	Model			Text					Yes
3)	File description	Text, describes file to load		Yes
4)	File to load		Optional path name and file name 	Yes
5)	Loader exe name		OPtional path name and file name 	Yes
		NOTE:	Both the "File to load" and "Loader exe name" Files MUST exist
			in the directory specified or if no directory is specified MUST
			exist in the SendGTL application directory or you will get
			error messages in the initial dialog box screen.

		NOTE: 	All the fields above are required, the fields
			below are optional. The	two above fields require
			a VALID file name with the optional path.
			The program DOES looks for the file and when the
			path is ommited the file is searched in the 
			application directory. That is where the
			SendGTL.EXE resides.
		NOTE:	In the comm settings fields below you can specify
			formatting parameters as well as the parameter itself
			for example to pass the comm port as a string as follows
			enter FORMAT_SPEC,DEFAULT, example: /com%i,
			use %i for an integer value, %f for a fractional number
			or %s for a string

[POSITION] [PARAMETER NAME]	[EXAMPLE]

6)	File to Load Format Specification, if any, eg. /f%s

7)	Comm Port		/com%i,1	for Comm Port defaulting to COM 1
8)	Baud Rate or BPS	/bps%i		without a default, user will select it
9)	Data Bits		/d%i,7		Default to seven data bits
10)	Stop Bits		/s%i
11)	Parity			/p%s		Use default parity from program, but format it
12)	Flow Control				BLANK, do not use field.
		NOTE:	If you do not want to use a field you MUST leave it blank,
			and the default parameter will be used,
			otherwise the it will be passed as a parameter.

13)	Extra Loader Parameters	Extra parameters passed to the application, No quotation marks
14)	Load File Version	Version information
15)	Extra User Instructions	User instructions displayed in the Select Dialog Box
16)	Comm Dialog BMP File	File name of a bitmap file (.BMP extension)
17)	Start Dialog BMP File	Same as above. This bitmaps replace the default pictures.

		NOTE: 	If you must specify a parameter that is optional,
			You must specify the previous parameters even if
			they are optionals OR you MUST put a | (pipe symbol)
			where the parameter would be. The parameters are searched
			according to their position. SendGTL uses default
			parameters for those ommited.
